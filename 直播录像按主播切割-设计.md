# 直播录像按主播切割 — 技术设计方案

## 1. 背景与目标

抖音带货直播录像，每场一个视频文件，长度数小时。一场内主播会换班，需要：
- 自动检测换班时间点
- 按换班点切割视频，每段归属一个主播
- 跨场次保持主播身份一致（主播 A 的所有段都进 `主播A/` 文件夹）

## 2. 关键约束

1. **开源、本地、零外发**
2. **跨场次主播一致性**：需要声纹库
3. **场景特征**：手播为主，无观众声，可能有助播/中控穿插话术
4. **不抽取主播独白**：助播插话原样保留在主播段中，只需要定位换班点
5. **过渡段丢弃**：换班交接前后 ±60s 丢弃
6. **无损切割 + 关键帧对齐**
7. **声纹库自动维护**：单场内连续 ≥2h 的 unknown 段自动入库为新主播
8. **段长过滤**：最终输出段必须 ≥1h，unknown 段全部丢弃

## 3. 问题建模

不是通用的 Speaker Diarization，而是 **"基于声纹库的主导说话人轨迹检测 + 变点检测"**：
- 库中已知主播 = 决定段归属的关键说话人
- 助播/中控 = 短时穿插的"噪声"，不应触发换班
- 输出：N 个换班点 → N+1 段视频

## 4. Pipeline 总览

```
视频文件
  ↓ ffmpeg 抽 16kHz mono wav
  ↓ Demucs 人声分离（去 BGM）
人声音轨
  ↓ 滑窗 60s/60s 提取 CAM++ 声纹（192 维）
embedding 序列
  ↓ 与声纹库 cosine 匹配 + 阈值/margin
逐窗标签序列（host_id / unknown）
  ↓ 90s 多数投票平滑（实际取 3 个窗口）
  ↓ 自动入库：连续 unknown ≥2h 时聚类，对每个 ≥1h 子簇都注册/合并 → 第二遍重标
  ↓ 合并相邻同标签段
  ↓ unknown 段 <30min 且两侧同主播 → 并入主播段
  ↓ 丢弃 unknown 段、丢弃 <1h 段
最终段列表 [(start, end, host_id), ...]
  ↓ ffprobe 取 I-frame，换班点向前对齐到最近关键帧
  ↓ ffmpeg -ss/-to -c copy 无损切割
归档目录 主播X/yyyymmdd_partN.mp4
```

## 5. 模块详细设计

### 5.1 预处理

- `ffmpeg -i input.mp4 -ac 1 -ar 16000 -vn out.wav`
- Demucs htdemucs 模型分离 vocals stem
- 输出：`vocals.wav`

### 5.2 滑窗声纹提取

- 窗口 60s，步长 60s（适配 3-5h 排班场景；窗口足够长让助播短插话不会主导嵌入）
- 模型：3D-Speaker CAM++（从 ModelScope 拉，本地缓存）
- 备选：ERes2Net / WeSpeaker ResNet34
- 流式 `soundfile.SoundFile.seek/read`，避免一次性把整段音频载入内存
- 支持 batch 推理，默认 batch size = 1；4GB GT 1030 可在空闲显存充足时尝试 2
- 输出：`N × 192` embedding 矩阵 + 时间戳序列

### 5.3 声纹库匹配

- 库结构：
  ```
  host_db/
    hosts.json          # {host_id, centroid_path, registered_at, samples_count, total_duration_h}
    host_001/
      centroid.npy
      samples/          # 历史样本 embedding，用于更新 centroid
  ```
- 每个窗口对所有 host centroid 计算 cosine sim
- 判定规则：
  - `max_sim > 0.65` 且 `top1 - top2 > 0.10` → 标 top1 host_id
  - 否则 → `unknown`
- 阈值需要按实际数据校准

### 5.4 平滑

- 90 秒滑动多数投票（在 60s 步长下按奇数滤波核取 3 个窗口），消除助播短插话造成的跳变

### 5.5 自动入库（两遍机制）

**第一遍**：用现有库做匹配 + 平滑，找连续 unknown 段
- 段长 <2h → 保持 unknown
- 段长 ≥2h →
  - 段内做 AgglomerativeClustering（cosine 距离阈值 0.25，average linkage）
  - 对每个子簇判断**占该 unknown 段的等效时长** `frac × duration`
    - 等效时长 ≥ `min_segment_sec`（1h）的子簇都尝试入库
    - 这样一段同时跨两个主播的 unknown（如 3h 内换班一次）可以一次注册两人，不要求"单簇占比 ≥80%"
  - 每个待入库簇先与现有 hosts 比 cosine：
    - max_sim > 0.75 → 视为已有主播，EMA 更新其 centroid
    - 否则 → 新建 host_N，写入库

**第二遍**：用更新后的库重跑匹配 + 平滑（解决第一遍因为库不全而误判 unknown 的段）

### 5.6 段合并与过滤

按以下顺序处理：
1. 合并相邻同 host_id 段
2. 对每个 unknown 段：若长度 <30min 且两侧是同一已知 host → 并入该 host
3. 丢弃所有 unknown 段
4. 丢弃所有长度 <1h 的段
5. 在剩余段之间引入 ±60s 过渡丢弃区

### 5.7 关键帧切割

- `ffprobe -show_frames -select_streams v -skip_frame nokey` 取所有 I-frame 时间戳
- 每个换班点向前对齐到最近关键帧
- `ffmpeg -ss <kf_time> -to <kf_time> -i input.mp4 -c copy -avoid_negative_ts make_zero`
- 输出到 `output/主播X/<原文件名>_partN.mp4`

### 5.8 声纹库更新策略

- 主播每被识别一段 ≥1h 的新片段后：
  - 该段 embedding 平均向量 → 加入 samples
  - centroid 用指数移动平均更新：`new = 0.9*old + 0.1*sample`（防漂移）
- 冲突检测：新主播入库前先与所有现有 centroid 比对，max_sim > 0.75 则合并而非新建

## 6. 关键参数

| 参数 | 默认值 | 备注 |
|---|---|---|
| 采样率 | 16 kHz mono | 标准声纹输入 |
| 窗口/步长 | 60s / 60s | 适配 3-5h 排班 & 助播短插话 |
| 声纹模型 | CAM++ | `iic/speech_campplus_sv_zh-cn_16k-common` |
| embedding 维度 | 192 | 与当前 CAM++ checkpoint 输出一致 |
| batch size | 1 | GT 1030 4GB DDR4 默认保守；显存充足时可尝试 2 |
| sim 阈值 | 0.65 | 识别归属 |
| margin 阈值 | 0.10 | top1 与 top2 差距 |
| 平滑窗口 | 90s（实际取 3 窗口） | 多数投票 |
| 自动入库时长 | ≥2h | 单场连续 unknown |
| 入库簇最小等效时长 | ≥1h | `frac × span_duration`；与 `min_segment_sec` 共用 |
| 入库聚类距离阈值 | 0.25 cosine | AgglomerativeClustering average linkage |
| 合库冲突阈值 | 0.75 | 比识别阈值严格 |
| unknown 合并阈值 | <30min | 主播短暂离场场景 |
| 过渡区丢弃 | ±60s | 交接寒暄；与 60s 步长精度匹配 |
| 最小段长 | 1h | 低于丢弃 |
| centroid EMA 权重 | 0.9 / 0.1 | 旧 / 新 |

## 7. 边界情况

| 场景 | 处理 |
|---|---|
| 主播 A 4h，中间助播多次插话 | 全归 A，1 个 4h 段 |
| 主播 A 3h → B 3h | 2 段，过渡区丢弃 |
| A 50min → B 50min | 都 <1h，全部丢弃 |
| A 3h → 助播 2.5h → A 1h | 助播段 ≥2h，聚类后等效时长 ≥1h 的子簇入库为新主播或合并到已有；若无 ≥1h 子簇 → unknown 被丢，两侧 A 段中间 >30min 不合并，输出 3h + 1h 两段 A |
| 库为空（冷启动） | 第一场所有窗口 unknown，整段 ≥2h；聚类得到的 ≥1h 子簇都会被注册（一段 3h 内换班 1 次可一次性入两人）|
| 两主播声音相似 | margin 检查导致大量 unknown，需人工提供更长 enrollment 样本 |

## 8. 技术栈

```
ffmpeg / ffprobe        视频处理
demucs                  人声分离
modelscope + 3D-Speaker 声纹模型
numpy / scipy           数值处理、滤波、聚类
scikit-learn            AgglomerativeClustering
```

全部本地、开源。GPU 推荐 NVIDIA + CUDA；CPU 可跑但慢约 10-15 倍。

## 9. 代码结构

```
pipeline/
  preprocess.py    # ffmpeg + demucs
  embedding.py     # CAM++ 滑窗提取
  matching.py      # 库匹配 + 平滑
  enrollment.py    # 自动入库 + 聚类
  segmenting.py    # 变点检测 + 段合并过滤
  cutting.py       # 关键帧对齐 + ffmpeg 切割
  db.py            # 声纹库读写、centroid 更新
  main.py          # CLI 入口
```

CLI:
```
python -m pipeline.main <video_path> [--dry-run] [--demucs] [--batch-size 1] [--db-dir host_db] [--output-dir output]
```

`--dry-run`：只输出换班点和归属预测，**不切视频**。

> ⚠️ 注意：`--dry-run` 仍会触发自动入库写入 `host_db/`（这是预测链路的必经环节，第二遍匹配需要新主播参与）。如果只想要"纯只读"预测，请先把 `host_db/` 备份或用 `--db-dir` 指向临时目录。
